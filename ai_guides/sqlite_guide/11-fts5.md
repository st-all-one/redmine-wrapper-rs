# Full-Text Search (FTS5)

Baseado em: `fts5.html`

---

## 1. O que é FTS5?

FTS5 é o módulo de Full-Text Search do SQLite, sucessor do FTS3/FTS4.
Permite busca textual eficiente com suporte a stemming, unicode,
classificação por relevância (BM25), e consultas booleanas.

### Principais Características

- Busca por palavras e frases
- Ranking por relevância (BM25)
- Stemming via tokenizers customizáveis
- Prefix queries
- Highlighting de termos
- Suporte a unicode61 (desde v3.41.0 como tokenizer nativo)
- Índices incrementais

## 2. Criação de Tabela FTS5

### Sintaxe Básica

```sql
CREATE VIRTUAL TABLE docs USING fts5(
    titulo,
    corpo,
    content='',           -- external content (opcional)
    content_rowid='rowid', -- external content rowid
    tokenize='porter unicode61',  -- tokenizer
    prefix='2,3',         -- prefix indexes (2-gram, 3-gram)
    detail='full'         -- full|column|none
);
```

### Tokenizers

| Tokenizer | Descrição |
|---|---|
| `unicode61` | Unicode-aware, remove acentos (default se omitido) |
| `ascii` | ASCII-only, separa por whitespace |
| `porter` | Stemmer de Porter (ex: "running" → "run") |
| `trigram` | Trigramas (para substring/fuzzy match) |

Os tokenizers podem ser combinados:
```sql
tokenize='porter unicode61'  -- stemmer + unicode
```

### Exemplos de Criação

```sql
-- FTS5 básico
CREATE VIRTUAL TABLE artigos USING fts5(
    titulo, conteudo, autor,
    tokenize='porter unicode61'
);

-- Com content externo (não duplica dados)
CREATE TABLE artigos_raw (
    id INTEGER PRIMARY KEY,
    titulo TEXT,
    conteudo TEXT,
    autor TEXT
);

CREATE VIRTUAL TABLE artigos_fts USING fts5(
    titulo, conteudo, autor,
    content='artigos_raw',
    content_rowid='id'
);

-- Triggers para manter sincronizado
CREATE TRIGGER artigos_ai AFTER INSERT ON artigos_raw BEGIN
    INSERT INTO artigos_fts(rowid, titulo, conteudo, autor)
    VALUES (new.id, new.titulo, new.conteudo, new.autor);
END;

CREATE TRIGGER artigos_ad AFTER DELETE ON artigos_raw BEGIN
    INSERT INTO artigos_fts(artigos_fts, rowid, titulo, conteudo, autor)
    VALUES ('delete', old.id, old.titulo, old.conteudo, old.autor);
END;

CREATE TRIGGER artigos_au AFTER UPDATE ON artigos_raw BEGIN
    INSERT INTO artigos_fts(artigos_fts, rowid, titulo, conteudo, autor)
    VALUES ('delete', old.id, old.titulo, old.conteudo, old.autor);
    INSERT INTO artigos_fts(rowid, titulo, conteudo, autor)
    VALUES (new.id, new.titulo, new.conteudo, new.autor);
END;
```

## 3. Consultas

### MATCH Operator

```sql
-- Palavra exata
SELECT * FROM artigos WHERE artigos MATCH 'SQLite';

-- Múltiplas palavras (AND implícito)
SELECT * FROM artigos WHERE artigos MATCH 'banco dados';
-- Equivalente a: 'banco AND dados'

-- OR explícito
SELECT * FROM artigos WHERE artigos MATCH 'SQLite OR MySQL';

-- NOT
SELECT * FROM artigos WHERE artigos MATCH 'SQLite -Oracle';

-- Frase exata
SELECT * FROM artigos WHERE artigos MATCH '"banco de dados"';

-- Prefix query
SELECT * FROM artigos WHERE artigos MATCH 'sql*';

-- Coluna específica
SELECT * FROM artigos WHERE artigos MATCH 'titulo:SQLite';
SELECT * FROM artigos WHERE artigos MATCH '(titulo:SQLite) OR (conteudo:MySQL)';

-- NEAR (próximo)
SELECT * FROM artigos WHERE artigos MATCH 'SQLite NEAR/5 dados';
```

### Funções Auxiliares

```sql
-- Rank por BM25 (default)
SELECT *, bm25(artigos) AS rank
FROM artigos
WHERE artigos MATCH 'SQLite'
ORDER BY rank;

-- Highlight (marca termos com HTML <b>)
SELECT highlight(artigos, 1, '<b>', '</b>') AS snippet
FROM artigos
WHERE artigos MATCH 'SQLite';

-- Snippet (fragmento contextual)
SELECT snippet(artigos, 1, '...', '...', '...', 32)
FROM artigos
WHERE artigos MATCH 'SQLite';

-- Contagem de ocorrências
SELECT *, rank.matches FROM artigos WHERE artigos MATCH 'SQLite';
```

### Parâmetros do BM25

```sql
-- BM25 com k1=1.2, b=0.75 (padrão é 1.0, 1.0)
SELECT *, bm25(artigos, 1.2, 0.75) AS rank
FROM artigos
WHERE artigos MATCH 'SQLite'
ORDER BY rank;
```

## 4. Manutenção do Índice

### Inserção e Deleção

```sql
-- Inserir (automático via content table)
INSERT INTO artigos_fts(rowid, titulo, conteudo, autor)
VALUES (1, 'Título', 'Conteúdo', 'Autor');

-- Deletar
INSERT INTO artigos_fts(artigos_fts, rowid, titulo, conteudo, autor)
VALUES ('delete', 1, 'Título', 'Conteúdo', 'Autor');

-- Command interface (via 'optimize')
INSERT INTO artigos_fts(artigos_fts) VALUES('optimize');
-- Reorganiza o índice (recomendado após muitas inserções/deleções)

-- Rebuild completo
INSERT INTO artigos_fts(artigos_fts) VALUES('rebuild');
```

### Merge de Segmentos

O FTS5 mantém segmentos (como LSM-tree). O merge automático ocorre,
mas pode ser controlado:

```sql
-- Configurar merge automático
-- Parâmetro: (-R, -I, ..., N) controla política de merge
INSERT INTO artigos_fts(artigos_fts, rank)
VALUES ('merge', 100);  -- merge de até 100 segmentos

-- Desabilitar merge automático (útil para bulk insert)
-- Depois otimizar manualmente
```

### Bulk Insert Otimizado

```sql
-- Para inserções em massa, desabilitar merge automático
PRAGMA synchronous = NORMAL;
BEGIN;

-- Desabilitar merge via command
INSERT INTO artigos_fts(artigos_fts) VALUES('automerge=0');

-- Inserir aos milhares
INSERT INTO artigos_fts(rowid, titulo, conteudo, autor)
VALUES (1, 'a', 'b', 'c'), (2, 'd', 'e', 'f'), ...;

-- Reabilitar e merge tudo
INSERT INTO artigos_fts(artigos_fts) VALUES('automerge=2');
INSERT INTO artigos_fts(artigos_fts) VALUES('merge=1000');

COMMIT;
```

## 5. Dicas de Performance

### Índices de Prefixo

```sql
-- Prefix indexes aceleram prefix searches como 'sql*'
CREATE VIRTUAL TABLE docs USING fts5(
    content,
    tokenize='porter unicode61',
    prefix='2,3'  -- índices para prefixos de 2 e 3 caracteres
);
-- Tamanho extra no índice: ~10-15%
-- Ganho: prefix queries ficam O(log N) em vez de O(N)
```

### detail (Controle de Armazenamento)

```sql
-- full (default): armazena posição + coluna de cada termo
CREATE VIRTUAL TABLE t1 USING fts5(a, b, detail='full');

-- column: só coluna, não posição (economia de ~40%)
CREATE VIRTUAL TABLE t2 USING fts5(a, b, detail='column');
-- Não suporta: frases, NEAR queries
-- Suporta: OR, AND, NOT, coluna específica

-- none: sem posição nem coluna (economia de ~50%)
CREATE VIRTUAL TABLE t3 USING fts5(a, b, detail='none');
-- Não suporta: frases, NEAR, coluna específica
-- Suporta: OR, AND, NOT
```

### Contentless vs External Content

```sql
-- content='' (contentless): só índice, sem dados
CREATE VIRTUAL TABLE t1 USING fts5(a, b, content='');
-- INSERT fornece dados + índice
-- SELECT só retorna rowid
-- UPDATE/DELETE não funcionam (precisa usar INSERT with 'delete')

-- content='tabela' (external): dados em outra tabela
CREATE VIRTUAL TABLE t2 USING fts5(a, b, content='tabela_real', content_rowid='id');
-- SELECT consulta a tabela real para obter dados
-- INSERT/UPDATE/DELETE só atualizam o índice (precisa triggers)
```

## 6. Trigram Tokenizer (Substring Search)

```sql
-- Trigram: permite busca por substring (LIKE '%termo%')
CREATE VIRTUAL TABLE busca USING fts5(
    texto,
    tokenize='trigram'
);

-- Consulta: busca substring
SELECT * FROM busca WHERE busca MATCH '"abc"';
-- Encontra qualquer texto contendo "abc"

-- Combinado: precisa de pelo menos 3 caracteres
SELECT * FROM busca WHERE busca MATCH '"sql"';
-- Encontra "SQLite", "MySQL", "sql", etc.
```

## 8. FTS5 vs LIKE

| Operação | FTS5 | LIKE '%termo%' |
|---|---|---|
| Palavra exata | `MATCH 'termo'` | `LIKE '%termo%'` |
| Múltiplas palavras | `MATCH 'a b'` (AND) | Múltiplos LIKE |
| Frase | `MATCH '"a b c"'` | `LIKE '%a b c%'` |
| Stemming | `MATCH 'running'` (acha "run") | `LIKE '%running%'` |
| Relevância | `bm25()` | Não disponível |
| Performance | O(log N) por termo | O(N) full scan |
| Tabela ~1M linhas | ~10ms | ~1000ms |

Sempre use FTS5 para busca textual. LIKE é adequado apenas para
tabelas muito pequenas (< 10k linhas).
